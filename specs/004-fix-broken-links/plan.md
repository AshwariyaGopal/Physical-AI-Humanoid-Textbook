# Implementation Plan - Fix Broken Links & Build Errors

**Branch**: `004-fix-broken-links`
**Spec**: [specs/004-fix-broken-links/spec.md](spec.md)

## Problem Analysis

The Vercel build is failing because of broken links. The error log indicates 6 specific broken links from the homepage (`/`) to module documentation pages.
Investigation reveals a mismatch between the **links** (which use the Mixed-Case filename, e.g., `Week-1-ROS2-Core-Concepts`) and the **actual routes** generated by Docusaurus.
The Markdown files contain frontmatter with lowercase `id` fields (e.g., `id: week-1-ros2-core-concepts`). Docusaurus uses this `id` to generate the route path, overriding the filename.
Therefore, the route is `/docs/Module-X.../week-1...` but the link points to `/docs/Module-X.../Week-1...`.

## Proposed Solution

Update all internal links in the codebase to match the actual route paths defined by the Docusaurus `id` frontmatter.
